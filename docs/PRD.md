# PRD｜北部工程排程＋庫存管理系統 v1.0

## 0. 專案定位

本系統是給北部工程內部使用的獨立工具，主要解決兩件事：

1. 排程管理
   - 多人任務排程
   - 案場任務安排
   - 週會展開報告
   - Google 共用行事曆同步

2. 庫存管理
   - 入庫 / 出庫 / 退料 / 調整
   - 掛鉤案場
   - 序號可補登
   - 月結報表

案場名冊作為共同基礎資料，排程與庫存都會掛鉤案場。

## 4. 案場名冊模組

### 4.1 案場資料用途

案場資料會被以下功能使用：

```text
排程任務 → 掛案場
庫存出庫 → 掛案場
退料紀錄 → 掛案場
Google Maps → 開啟地址
週報展開 → 顯示案場
案場頁 → 查任務與用料
```

### 4.2 案場欄位

| 欄位 | 說明 | 必填 |
|---|---|---|
| 案場名稱 | 完整案場名稱 | 是 |
| 案場簡稱 | 排程卡片顯示用 | 否 |
| 地址 | Google Maps 使用 | 否 |
| 業主 | 可選填 | 否 |
| 聯絡人 | 可選填 | 否 |
| 聯絡電話 | 可選填 | 否 |
| 備註 | 可選填 | 否 |
| 是否啟用 | 停用後不出現在常用選單 | 是 |

### 4.3 案場新增方式

系統需支援三種新增方式。

#### A. 案場名冊頁新增

完整建立案場資料。

#### B. 新增排程時快速新增

當排程找不到案場時，可快速新增：

```text
案場名稱
案場簡稱
地址
```

#### C. 出庫 / 退料時快速新增

當庫存紀錄找不到案場時，可快速新增：

```text
案場名稱
案場簡稱
地址
```

### 4.4 案場匯入

第一階段支援：

```text
Google Sheet → 匯出 Excel / CSV → 匯入系統
```

暫不做 Google Sheet API 即時同步。

### 4.5 Google Maps

第一階段使用外部 Google Maps URL。

```text
https://www.google.com/maps/search/?api=1&query=地址
```

案場頁與任務詳情頁都要有：

```text
[開啟 Google Maps]
```

## 5. 排程系統

### 5.1 排程核心概念

排程是多人共用，不是每個人一個獨立行事曆。

一個任務可以有：

```text
主要負責人
協同人員
```

同一個任務只顯示一張卡，不因為多人參與就重複顯示。

### 5.2 排程檢視

系統需提供以下檢視。

```text
週曆檢視
月曆檢視
當日展開檢視
週報展開檢視
待辦事項
```

手機版後續再細談，第一階段先確保基本可用。

## 6. 週曆檢視

### 6.1 週曆設計原則

週曆主畫面要乾淨，不顯示固定時間段。

不要做：

```text
08:00-10:00
10:00-12:00
13:00-15:00
15:00-17:00
上午
下午
全天
未定
```

改成：

```text
一週七天欄位
每天底下依任務開始時間排序
```

### 6.2 週曆主畫面範例

```text
          一 6/8          二 6/9          三 6/10         四 6/11         五 6/12

          [送電] 泰電      [現勘] 聯華      [維修] 神行
          柚/竹            柚              揚

          [維修] 神行      [暫][施工] 威策
          揚               柚/竹

          [確認] Jeff
          柚

          +2
```

### 6.3 任務排序規則

同一天內，任務排序如下：

| 順序 | 規則 |
|---|---|
| 1 | 有開始時間者，依開始時間由早到晚 |
| 2 | 全天任務 |
| 3 | 沒有時間的任務 |
| 4 | 暫定任務不另外分區，照時間排序 |

### 6.4 每日顯示數量

每一天最多直接顯示 3 筆任務。

超過顯示：

```text
+2
```

點擊日期或 `+N` 後，展開當日完整任務清單。

### 6.5 任務卡片格式

週曆主畫面卡片只顯示：

```text
[類型] 案場簡稱
人員簡稱
```

範例：

```text
[送電] 泰電
柚/竹
```

```text
[維修] 神行
揚
```

```text
[暫][送電] 泰電
柚/竹
```

完成後：

```text
✓ [送電] 泰電
柚/竹
```

並且整張卡片變淡。

### 6.6 任務顏色

任務顏色依「任務類型」區分，不依人員區分。

建議類型：

| 類型 |
|---|
| 現勘 |
| 維修 |
| 施工 |
| 掛表 |
| 送電 |
| 清洗 |
| 電檢 |
| 確認 |
| 內部 |
| 其他 |

## 10. 任務新增與編輯

### 10.1 任務欄位

| 欄位 | 說明 | 必填 |
|---|---|---|
| 任務類型 | 現勘 / 維修 / 施工 / 掛表 / 送電等 | 是 |
| 任務標題 | 可簡短描述事項 | 是 |
| 案場 | 可選既有案場，也可快速新增 | 否 |
| 日期 | 任務日期 | 是 |
| 開始時間 | 可不填 | 否 |
| 結束時間 | 預設開始時間 +2 小時，可手動改 | 否 |
| 全天 | 可勾選 | 否 |
| 暫定 | 可勾選 | 否 |
| 主要負責人 | 任務主要負責人 | 是 |
| 協同人員 | 可多人 | 否 |
| 地址 | 選案場後自動帶入 | 否 |
| 備註 | 可輸入詳細內容 | 否 |
| 狀態 | 未開始 / 進行中 / 已完成 / 取消 | 是 |
| 是否同步 Google Calendar | 預設同步 | 否 |

### 10.2 時間邏輯

#### 情境 A：有開始時間

使用者填：

```text
09:00
```

系統自動預設：

```text
開始時間：09:00
結束時間：11:00
```

使用者可手動改結束時間。

#### 情境 B：全天

使用者勾選全天。

任務不需要開始 / 結束時間。

#### 情境 C：未填時間

只選日期，不填時間。

系統視為：

```text
未指定時間
```

週曆主畫面仍顯示在該日欄位內。

### 10.3 暫定任務

暫定任務顯示：

```text
[暫][送電] 泰電
柚/竹
```

暫定任務仍可：

- 填日期
- 填時間
- 掛案場
- 派人員
- 顯示在週報

### 10.4 完成任務

任務狀態改為已完成後：

- 任務卡變淡
- 可顯示 `✓`
- 仍保留在週曆與週報中

## 11. 衝突提醒

### 11.1 衝突判斷

新增或拖曳任務時，系統需檢查：

```text
同一天
同一人員
時間重疊
狀態不是取消
```

### 11.2 衝突處理

只提醒，不阻擋。

範例：

```text
此時段已有其他任務：

09:00-11:00 [送電] 泰電｜柚/竹
09:30-11:30 [維修] 神行｜柚

仍要建立嗎？

[仍然建立] [返回修改]
```

### 11.3 不算硬衝突的情況

| 情況 | 處理 |
|---|---|
| 不同人員時間重疊 | 不提醒 |
| 全天任務 | 可提示，但不視為硬衝突 |
| 未指定時間 | 不算衝突 |
| 暫定任務 | 可提醒為暫定衝突 |

## 12. 拖曳排程

### 12.1 拖曳功能

系統排程需支援拖曳。

| 操作 | 結果 |
|---|---|
| 拖到其他日期 | 改日期 |
| 拖到其他時間 | 改開始 / 結束時間 |
| 拖曳後有衝突 | 顯示提醒，不阻擋 |
| 拖曳後同步 Google Calendar | 需要 |

### 12.2 Google Calendar 同步流程

```text
系統內拖曳任務
↓
更新本系統任務資料
↓
呼叫 Google Calendar API
↓
更新共用行事曆事件
↓
顯示同步成功 / 失敗
```

如果同步失敗：

```text
本系統已更新，但 Google Calendar 同步失敗。
[重新同步]
```

## 14. 待辦事項

### 14.1 目的

待辦事項要簡單，不做完整工單系統。

用途：

- 先記錄事情
- 讓別人申請事項
- 之後可順手轉成正式排程

### 14.2 待辦欄位

| 欄位 | 說明 | 必填 |
|---|---|---|
| 標題 | 待辦事項名稱 | 是 |
| 內容 | 可輸入詳細資訊 | 否 |
| 案場 | 可選，可快速新增 | 否 |
| 任務類型 | 可選 | 否 |
| 建立人 | 自動帶入 | 是 |
| 狀態 | 待安排 / 已排程 / 取消 | 是 |

### 14.3 待辦畫面

範例：

```text
待辦事項

[+] 新增待辦

- 神行欣業 S1200
  要更換 16 顆，等確認料況

- 泰電第八台測試
  週末較適合，需 Jeff 確認產線

- 沛亨水管
  先接好後再與業主討論
```

### 14.4 待辦轉排程

每筆待辦可按：

```text
[排入行程]
```

點擊後開啟新增任務視窗，自動帶入：

- 標題
- 內容
- 案場
- 任務類型

## 15. 庫存系統

### 15.1 庫存定位

目前只有一個實體倉庫，但庫存需分成三個類別。

不要用三個倉庫概念，而是用「庫存類別」。

### 15.2 庫存類別

固定三類：

| 類別 | 說明 |
|---|---|
| 建置物料 | 新建案場施工用 |
| 維修備品 | 北部工程常備維修用 |
| 廠商備品 | 設備廠商提供之維修品 |

廠商名稱可記錄，但不作為主要追蹤重點。

### 15.3 品項管理

| 欄位 | 說明 | 必填 |
|---|---|---|
| 品項名稱 | 例如 S1200 優化器 | 是 |
| 品項代碼 | 可自動產生 | 是 |
| 分類 | 優化器 / 逆變器 / 線材 / 電料 / 五金 / 其他 | 是 |
| 單位 | 顆 / 台 / 米 / 組 / 箱 | 是 |
| 是否需要序號 | 是 / 否 | 是 |
| 安全庫存 | 低於提醒，第二階段使用 | 否 |
| 是否啟用 | 是 / 否 | 是 |
| 備註 | 可選填 | 否 |

### 15.4 線材管理

線材單位用：

```text
米
```

第一階段不管理整捲、捲號、餘料卷。

只管理：

```text
入庫幾米
出庫幾米
剩餘幾米
```

## 16. 庫存異動類型

庫存流水帳需支援四種異動。

| 類型 | 說明 | 庫存影響 |
|---|---|---|
| IN | 入庫 | 增加 |
| OUT | 出庫 | 減少 |
| RETURN | 退料 | 增加 |
| ADJUST | 盤點調整 | 增加或減少 |

所有庫存變動都必須寫入流水帳，不直接修改總庫存。

## 19. 序號管理

### 19.1 核心需求

本次序號管理只需做到：

```text
東西到我這裡
後續拿哪個序號去案場維修
可以查得到
```

不做複雜 RMA / 保固 / 故障履歷。

### 19.2 序號狀態

| 狀態 | 說明 |
|---|---|
| 在庫 | 目前在倉庫 |
| 已出庫 | 已拿去案場 |
| 已使用 | 已用於案場 |
| 已退回 | 從案場退回 |
| 待補 | 這筆出庫尚未補序號 |
| 報廢 | 不可再使用 |

### 19.3 序號可補登

出庫時若太忙，允許先不填序號。

例如：

```text
品項：S1200
數量：2
案場：神行欣業
序號狀態：待補
```

系統需記錄：

```text
此筆出庫尚有 2 個序號待補
```

之後可回來補：

```text
SN123
SN124
```

### 19.4 待補序號清單

系統需有一頁或一區顯示：

```text
待補序號
```

欄位：

| 欄位 | 說明 |
|---|---|
| 日期 | 出庫日期 |
| 案場 | 使用案場 |
| 品項 | 品項名稱 |
| 數量 | 待補數量 |
| 登記人 | 誰登記 |
| 操作 | 補登序號 |

## 22. 月結功能

### 22.1 月結目標

每月可產生庫存結算：

- 月初庫存
- 本月入庫
- 本月出庫
- 本月退料
- 本月調整
- 月底庫存

### 22.2 月結欄位

| 欄位 | 說明 |
|---|---|
| 月份 | 例如 2026-06 |
| 品項 | 品項名稱 |
| 庫存類別 | 建置物料 / 維修備品 / 廠商備品 |
| 月初庫存 | 上月月底 |
| 本月入庫 | IN |
| 本月出庫 | OUT |
| 本月退料 | RETURN |
| 本月調整 | ADJUST |
| 月底庫存 | 月初 + 入庫 - 出庫 + 退料 + 調整 |
| 單位 | 顆 / 台 / 米 |
| 備註 | 可填 |

### 22.3 月結規則

第一階段：

```text
產生報表，不鎖帳
```

也就是：

- 可產生月結
- 可重新產生
- 若資料補登，月結重新計算

第二階段再做：

```text
正式結帳
鎖定月份
Admin 解鎖
```
