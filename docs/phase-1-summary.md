# Phase 1: Management System Refinement 總結

## 1. 排程管理目前功能
* 提供日曆檢視模式，可透過拖曳方式調整排程時間。
* 支援多種任務類型（例如：安裝、維修、場勘等）。
* 任務建立後可指定主要負責人與協辦人員。
* POC 階段：已完成 Supabase 基本連線與 `schedule_tasks` 資料表串接測試，可即時新增、更新任務。

## 2. 案場管理目前功能
* 統一核心資料架構，將「所有案場」與「進行中案場」整合為單一資料來源 (`projects`)。
* 依據狀態（開案、施工中、待驗收、待掛表、已結案、作廢）區分顯示頁面。
* 支援詳細資料檢視與內聯修改（Inline Edit）。
* 已加入「包商排工」支援（支架、機電、鋼構、浪板、土木、其他等 6 種固定工種）。
* 具備完整聯絡人、地址與保固狀態紀錄。

## 3. 進行中案場周回報目前功能
* 保留原有的四區顯示邏輯（施工中、預計進場、掛表前、其他），不破壞原有閱讀習慣。
* 支援右鍵選單，可快速進行「詳細資料」、「改期」、「結案」、「作廢 / 停用」等操作。
* 已實作重構後的詳細資料彈窗（共分四個分頁：基本資料、包商資料、施工進度、備註）。

## 4. 庫存管理目前功能
* 支援出入庫流水紀錄與分類。
* 可自動計算目前庫存餘額。
* 支援月結功能，能產生月結報表與快照。

## 5. SE 供貨目前功能
* 獨立追蹤特定 SE 設備（如逆變器等）的供貨狀態。
* 包含預計到貨、實際到貨、派送日期等欄位，方便與案場排程對接。

## 6. 人員管理目前功能
* 提供人員名單與角色權限設定（管理員、一般使用者、檢視者）。
* 新增「類別」設定，區分工程部人員（ENGINEERING）與其他人員。
* 提供停用（Inactive）功能，保留歷史紀錄。

## 7. 尚未完成 / 待後續處理事項
* 尚未正式將所有資料庫模組（Project、Inventory 等）完整搬移至 Supabase，目前僅完成排程的 POC。
* 「包商排工」的甘特圖視覺化尚未實作完成。
* 改期功能的完整聯動（例如拖曳後同步發送通知或自動排開）尚未完成。
* 用戶真實登入驗證（Login Authentication）。
* 資料表之間更強的關聯性索引與約束建置。

## 8. 下一階段建議：資料庫、登入權限、Google Calendar
* **資料庫：** 建議根據目前的 POC 經驗，全面採用 Supabase 作為系統後端，並設計完整的 Migration 計畫，將 Mock Data 全部轉移至 PostgreSQL。
* **登入權限：** 導入 Supabase Auth 或是 NextAuth，將目前的 `currentUser` mock 替換為真實的 JWT token 驗證，並在資料庫層級加上 RLS（Row Level Security）保護檢視者權限。
* **Google Calendar 整合：** 建議透過 Google API 進行 OAuth 授權，讓排程系統能夠與使用者的 Google Calendar 進行雙向或單向同步，減少重複建檔的時間成本。
