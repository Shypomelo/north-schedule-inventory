# Phase 1 管理系統階段性總結

本文件記錄「北部工程排程與庫存管理系統」第一階段（Phase 1）完成的各項核心模組功能，以及尚未完成與未來下一階段的重點規劃。

## 1. 模組功能現況

### 排程管理目前功能
* **日曆檢視**：提供直覺的週視角，可以快速拖曳（Drag & Drop）排程卡片來變更日期。
* **任務管理**：可建立、編輯、指派主要負責人與多位協同人員。任務包含全天與特定時段的設定。
* **待辦清單 (Todo)**：支援獨立的待辦清單，可拖曳至日曆中轉換為正式排程，亦可從排程退回待辦狀態。
* **狀態與防呆**：任務可標記為「暫定」、「完成」、「改期」等狀態；針對 VIEWER 權限已鎖定拖曳與右鍵編輯功能。

### 案場管理目前功能
* **所有案場 (Base)**：維護所有案場的基本資料、合約保固狀態與逆變器等硬體資訊。
* **進行中案場 (Active)**：從基礎案場中標記為「進行中」後進入，支援週回報。
* **人員專屬視角**：側邊欄會動態依據「工程人員」產生專屬的「某某案場」過濾頁面。

### 進行中案場周回報目前功能
* **行內編輯 (Inline Edit)**：在表格上直接點擊下拉或日曆，即可快速更新各項施工進度（如：支架、模組、檢驗、掛表等日期）。
* **狀態追蹤**：視覺化顯示「已完成」、「進行中」、「未開始」等狀態，並包含備註輸入。
* **結案操作**：完成所有階段後，可一鍵「結案」，將案場從進行中列表歸檔回所有案場。

### 庫存管理目前功能
* **品項總覽**：維護庫存品項，支援安全庫存（低庫存門檻）提示，支援「是否需要序號」的屬性。
* **異動紀錄與進出庫**：提供進貨、出庫、退料等操作，庫存數量自動加減。
* **序號追蹤**：針對需要序號的設備類品項，支援出入庫時的序號勾選與掃描補登，確保硬體溯源。
* **月結報表**：提供期初、期末與本期異動統計的歷史報表功能。

### SE 供貨目前功能
* **訂單追蹤**：管理 SE 相關供貨訂單。
* **狀態與出貨**：支援下單、預計到貨、實際到貨等狀態追蹤。
* **防呆保護**：對應的狀態變更與欄位編輯會受使用者權限限制。

### 人員管理目前功能
* **全域使用者資料庫**：統一人員管理介面，所有系統中的「人員選單」皆由此動態產生。
* **角色與權限**：區分 `ADMIN`、`ENGINEER`、`VIEWER`。VIEWER 在所有功能中皆只能檢視，無法編輯、儲存或拖曳。
* **人員分類與狀態**：可將人員分為「工程」或「其他」，並支援「啟用 / 停用」機制，停用後人員不會出現在新的選單中，但歷史紀錄不受影響。

---

## 2. 尚未完成 / 待後續處理事項

* **匯出與報表**：尚未實作將排程或庫存資料匯出為 Excel/PDF 的功能。
* **進階篩選與搜尋**：目前雖然有基礎搜尋，但跨模組的全域搜尋（例如用關鍵字找出所有相關的排程與庫存）尚未實作。
* **自動提醒與通知**：尚無 Email 或 Line 機器人串接來自動通知排程變更或低庫存。

---

## 3. 下一階段建議 (Phase 2)

### 1. 實際資料庫建置 (Real Database)
* 目前系統使用 `localStorage` 作為 Mock DB 以進行快速原型開發與驗證。
* **建議**：導入正式資料庫（如 PostgreSQL 或 MySQL）搭配 Prisma 等 ORM 進行後端 API 串接，確保資料永久儲存與多人同時操作的一致性。

### 2. 登入與身分驗證 (Authentication)
* 目前是透過 UI 下拉選單進行「暫時性使用者切換」。
* **建議**：整合 NextAuth.js 或其他驗證機制（如 Google OAuth 或帳號密碼登入），將使用者的 session 與真實身分綁定，從後端徹底鎖定權限。

### 3. Google Calendar 雙向同步
* 目前在排程資料結構中已有預留 `google_calendar_id` 與同步狀態欄位。
* **建議**：實作 Google Calendar API 串接，讓建立排程任務時自動同步到工程師的 Google 日曆上，並能接收日曆的變更。
